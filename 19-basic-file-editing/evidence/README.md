# vi Practice Evidence

This directory is reserved for sanitized evidence from completed vi exercises.

The strongest evidence for this interactive module is the Git commit history:

1. Commit the original sample configuration.
2. Edit the configuration locally using `vi`.
3. Commit the edited version separately.
4. Use the GitHub diff to show the exact changes made.

No credentials, production configuration, private email addresses, hostnames, or real server information should be committed.
